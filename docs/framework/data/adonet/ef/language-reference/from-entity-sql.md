### <a name="join-from-clause-item"></a><span data-ttu-id="3577c-122">JOIN FROM 句の項目</span><span class="sxs-lookup"><span data-stu-id="3577c-122">JOIN FROM Clause Item</span></span>

#### <a name="inner-joins"></a><span data-ttu-id="3577c-131">内部結合</span><span class="sxs-lookup"><span data-stu-id="3577c-131">Inner Joins</span></span>

 <span data-ttu-id="3577c-133">上記のクエリ式では、`ON` 条件が指定されており、右のコレクションの各要素と対になっている左のコレクションの各要素を結合して処理します。</span><span class="sxs-lookup"><span data-stu-id="3577c-133">The previous query expression processes a combination of every element of the collection on the left paired against every element of the collection on the right, where the `ON` condition is true.</span></span> <span data-ttu-id="3577c-134">`ON` 条件を指定しない場合、`INNER JOIN` の動作は `CROSS JOIN` と同じになります。</span><span class="sxs-lookup"><span data-stu-id="3577c-134">If no `ON` condition is specified, an `INNER JOIN` degenerates to a `CROSS JOIN`.</span></span>

 <span data-ttu-id="3577c-137">上記のクエリ式では、`ON` 条件が指定されており、右のコレクションの各要素と対になっている左のコレクションの各要素を結合して処理します。</span><span class="sxs-lookup"><span data-stu-id="3577c-137">The previous query expression processes a combination of every element of the collection on the left paired against every element of the collection on the right, where the `ON` condition is true.</span></span> <span data-ttu-id="3577c-138">`ON` 条件が指定されていない場合、式は、NULL 値を使用して、右の要素と対になっている左の要素の単一のインスタンスを処理します。</span><span class="sxs-lookup"><span data-stu-id="3577c-138">If the `ON` condition is false, the expression still processes a single instance of the element on the left paired against the element on the right, with the value null.</span></span>  
  
 <span data-ttu-id="3577c-139">`RIGHT OUTER JOIN` についても同様です。</span><span class="sxs-lookup"><span data-stu-id="3577c-139">A `RIGHT OUTER JOIN` may be expressed in a similar manner.</span></span>

 <span data-ttu-id="3577c-142">上記のクエリ式では、`ON` 条件が指定されており、右のコレクションの各要素と対になっている左のコレクションの各要素を結合して処理します。</span><span class="sxs-lookup"><span data-stu-id="3577c-142">The previous query expression processes a combination of every element of the collection on the left paired against every element of the collection on the right, where the `ON` condition is true.</span></span> <span data-ttu-id="3577c-143">`ON` 条件が指定されていない場合、式は、NULL 値を使用して、右の要素と対になっている左の要素のインスタンスを 1 つ処理します。</span><span class="sxs-lookup"><span data-stu-id="3577c-143">If the `ON` condition is false, the expression still processes one instance of the element on the left paired against the element on the right, with the value null.</span></span> <span data-ttu-id="3577c-144">また、NULL 値を使用して、左の要素と対になっている右の要素のインスタンスも 1 つ処理します。</span><span class="sxs-lookup"><span data-stu-id="3577c-144">It also processes one instance of the element on the right paired against the element on the left, with the value null.</span></span>

### <a name="apply-clause-item"></a><span data-ttu-id="3577c-147">APPLY 句の項目</span><span class="sxs-lookup"><span data-stu-id="3577c-147">APPLY Clause Item</span></span>  
 [!INCLUDE[esql](../../../../../../includes/esql-md.md)] <span data-ttu-id="3577c-148">は、`APPLY` と `CROSS APPLY` という 2 種類の `OUTER APPLY` をサポートしています。</span><span class="sxs-lookup"><span data-stu-id="3577c-148">supports two kinds of `APPLY`: `CROSS APPLY` and `OUTER APPLY`.</span></span>  
  
 <span data-ttu-id="3577c-149">`CROSS APPLY` は、左のコレクションの各要素と右の式の評価によって生成されたコレクションの要素との一意の組み合わせを生成します。</span><span class="sxs-lookup"><span data-stu-id="3577c-149">A `CROSS APPLY` produces a unique pairing of each element of the collection on the left with an element of the collection produced by evaluating the expression on the right.</span></span> <span data-ttu-id="3577c-150">`CROSS APPLY` では、右の式は左の要素に機能的に依存します。このような関連付けられたコレクションの例を次に示します。</span><span class="sxs-lookup"><span data-stu-id="3577c-150">With a `CROSS APPLY`, the expression on the right is functionally dependent on the element on the left, as illustrated in the following associated collection example:</span></span>

 <span data-ttu-id="3577c-151">`CROSS APPLY` の動作は結合リストに似ています。</span><span class="sxs-lookup"><span data-stu-id="3577c-151">The behavior of `CROSS APPLY` is similar to the join list.</span></span> <span data-ttu-id="3577c-152">右の式が空のコレクションとして評価される場合、`CROSS APPLY` は、左の要素のそのインスタンスの組み合わせを生成しません。</span><span class="sxs-lookup"><span data-stu-id="3577c-152">If the expression on the right evaluates to an empty collection, the `CROSS APPLY` produces no pairings for that instance of the element on the left.</span></span>

## <a name="multiple-collections-in-the-from-clause"></a><span data-ttu-id="3577c-160">FROM 句での複数のコレクション</span><span class="sxs-lookup"><span data-stu-id="3577c-160">Multiple Collections in the FROM Clause</span></span>

## <a name="left-correlation"></a><span data-ttu-id="3577c-166">左の相関関係</span><span class="sxs-lookup"><span data-stu-id="3577c-166">Left Correlation</span></span>

```  
from C as c, D as d, c.Names as e  
```  
  
 <span data-ttu-id="3577c-169">上記の例は、次の例と論理的に等価です。</span><span class="sxs-lookup"><span data-stu-id="3577c-169">This is logically equivalent to:</span></span>  
  
```  
from (C as c join D as d) cross apply c.Names as e  
```  
  
## <a name="semantics"></a><span data-ttu-id="3577c-170">Semantics</span><span class="sxs-lookup"><span data-stu-id="3577c-170">Semantics</span></span>  
 <span data-ttu-id="3577c-171">論理上、`FROM` 句のコレクションは `n` 方向のクロス結合の一部になると見なされます (1 方向のクロス結合の場合を除く)。</span><span class="sxs-lookup"><span data-stu-id="3577c-171">Logically, the collections in the `FROM` clause are assumed to be part of an `n`-way cross join (except in the case of a 1-way cross join).</span></span> <span data-ttu-id="3577c-172">`FROM` 句内の別名は、左から右へと処理され、後で参照できるように現在のスコープに追加されます。</span><span class="sxs-lookup"><span data-stu-id="3577c-172">Aliases in the `FROM` clause are processed left to right, and are added to the current scope for later reference.</span></span> <span data-ttu-id="3577c-173">`FROM` 句は、行のマルチセットを生成すると見なされます。</span><span class="sxs-lookup"><span data-stu-id="3577c-173">The `FROM` clause is assumed to produce a multiset of rows.</span></span> <span data-ttu-id="3577c-174">`FROM` 句の項目ごとに 1 つのフィールドが生成され、そのコレクションの項目の 1 つの要素を表します。</span><span class="sxs-lookup"><span data-stu-id="3577c-174">There will be one field for each item in the `FROM` clause that represents a single element from that collection item.</span></span>  
  
 <span data-ttu-id="3577c-175">`FROM` 句は、c、d、e の各フィールドが `C`、`D`、`c.Names` の要素型になると見なされる Row(c, d, e) 型の行のマルチセットを論理的に生成します。</span><span class="sxs-lookup"><span data-stu-id="3577c-175">The `FROM` clause logically produces a multiset of rows of type Row(c, d, e) where fields c, d, and e are assumed to be of the element type of `C`, `D`, and `c.Names`.</span></span>  
  
 [!INCLUDE[esql](../../../../../../includes/esql-md.md)] <span data-ttu-id="3577c-176">では、単純な `FROM` 句の項目ごとの別名がスコープに導入されます。</span><span class="sxs-lookup"><span data-stu-id="3577c-176">introduces an alias for each simple `FROM` clause item in scope.</span></span> <span data-ttu-id="3577c-177">たとえば、次の FROM 句の例では、スコープに導入される名前は c、d、e です。</span><span class="sxs-lookup"><span data-stu-id="3577c-177">For example, in the following FROM clause snippet, The names introduced into scope are c, d, and e.</span></span>  
  
```  
from (C as c join D as d) cross apply c.Names as e  
```  
  
 <span data-ttu-id="3577c-178">[!INCLUDE[esql](../../../../../../includes/esql-md.md)] では [!INCLUDE[tsql](../../../../../../includes/tsql-md.md)] と異なり、`FROM` 句では別名のみがスコープに導入されます。</span><span class="sxs-lookup"><span data-stu-id="3577c-178">In [!INCLUDE[esql](../../../../../../includes/esql-md.md)] (unlike [!INCLUDE[tsql](../../../../../../includes/tsql-md.md)]), the `FROM` clause only introduces the aliases into scope.</span></span> <span data-ttu-id="3577c-179">これらのコレクションの列 (プロパティ) への参照は、別名を使用して修飾する必要があります。</span><span class="sxs-lookup"><span data-stu-id="3577c-179">Any references to columns (properties) of these collections must be qualified with the alias.</span></span>  
  
## <a name="pulling-up-keys-from-nested-queries"></a><span data-ttu-id="3577c-180">入れ子になったクエリからのキーの抽出</span><span class="sxs-lookup"><span data-stu-id="3577c-180">Pulling Up Keys from Nested Queries</span></span>

 <span data-ttu-id="3577c-183">しかし、次のクエリは、入れ子になったクエリにキーがないため無効です。</span><span class="sxs-lookup"><span data-stu-id="3577c-183">However, the following query is not valid, because the nested query does not have any keys:</span></span>
