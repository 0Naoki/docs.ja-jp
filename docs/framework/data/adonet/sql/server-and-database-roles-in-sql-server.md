# <a name="server-and-database-roles-in-sql-server"></a><span data-ttu-id="e0397-102">SQL Server のサーバー ロールとデータベース ロール</span><span class="sxs-lookup"><span data-stu-id="e0397-102">Server and Database Roles in SQL Server</span></span>
<span data-ttu-id="e0397-103">SQL Server では、すべてのバージョンで、個々のユーザーではなくロール (つまり、ユーザーのグループ) に対して権限を割り当てることのできるロール ベースのセキュリティが使用されています。</span><span class="sxs-lookup"><span data-stu-id="e0397-103">All versions of SQL Server use role-based security, which allows you to assign permissions to a role, or group of users, instead of to individual users.</span></span> <span data-ttu-id="e0397-104">固定サーバー ロールおよび固定データベース ロールには、固定された一連の権限が割り当てられています。</span><span class="sxs-lookup"><span data-stu-id="e0397-104">Fixed server and fixed database roles have a fixed set of permissions assigned to them.</span></span>  
  
## <a name="fixed-server-roles"></a><span data-ttu-id="e0397-105">固定サーバー ロール</span><span class="sxs-lookup"><span data-stu-id="e0397-105">Fixed Server Roles</span></span>  
 <span data-ttu-id="e0397-106">固定サーバー ロールは、固定された一連の権限とサーバー規模のスコープを持ちます。</span><span class="sxs-lookup"><span data-stu-id="e0397-106">Fixed server roles have a fixed set of permissions and server-wide scope.</span></span> <span data-ttu-id="e0397-107">SQL Server の管理を目的としており、このロールに割り当てられた権限を変更することはできません。</span><span class="sxs-lookup"><span data-stu-id="e0397-107">They are intended for use in administering SQL Server and the permissions assigned to them cannot be changed.</span></span> <span data-ttu-id="e0397-108">固定サーバー ロールには、データベースにユーザー アカウントがなくても、ログインを割り当てることができます。</span><span class="sxs-lookup"><span data-stu-id="e0397-108">Logins can be assigned to fixed server roles without having a user account in a database.</span></span>  
  
> [!IMPORTANT]
>  <span data-ttu-id="e0397-109">`sysadmin` 固定サーバー ロールは他のすべてのロールを内包し、無制限のスコープを持ちます。</span><span class="sxs-lookup"><span data-stu-id="e0397-109">The `sysadmin` fixed server role encompasses all other roles and has unlimited scope.</span></span> <span data-ttu-id="e0397-110">本当に信頼できる場合以外は、このロールにプリンシパルを追加することは避けてください。</span><span class="sxs-lookup"><span data-stu-id="e0397-110">Do not add principals to this role unless they are highly trusted.</span></span> `sysadmin` <span data-ttu-id="e0397-111">ロールのメンバーでは、すべてのサーバーのデータベースおよびリソースに対する取り消し不可能の管理者権限があります。</span><span class="sxs-lookup"><span data-stu-id="e0397-111">role members have irrevocable administrative privileges on all server databases and resources.</span></span>  
  
 <span data-ttu-id="e0397-112">固定サーバー ロールに追加するユーザーは慎重に選ぶ必要があります。</span><span class="sxs-lookup"><span data-stu-id="e0397-112">Be selective when you add users to fixed server roles.</span></span> <span data-ttu-id="e0397-113">たとえば、`bulkadmin` ロールでは、ユーザーがローカル ファイルの内容をテーブルに挿入できるため、データの整合性が損なわれる可能性があります。</span><span class="sxs-lookup"><span data-stu-id="e0397-113">For example, the `bulkadmin` role allows users to insert the contents of any local file into a table, which could jeopardize data integrity.</span></span> <span data-ttu-id="e0397-114">固定サーバー ロールとアクセス許可の完全な一覧については、SQL Server オンライン ブックを参照します。</span><span class="sxs-lookup"><span data-stu-id="e0397-114">See SQL Server Books Online for the complete list of fixed server roles and permissions.</span></span>

## <a name="database-roles-and-users"></a><span data-ttu-id="e0397-125">データベース ロールおよびユーザー</span><span class="sxs-lookup"><span data-stu-id="e0397-125">Database Roles and Users</span></span>  
 <span data-ttu-id="e0397-126">データベース ユーザー アカウントはデータベース オブジェクトを扱う関係上、ログインにマップされている必要があります。</span><span class="sxs-lookup"><span data-stu-id="e0397-126">Logins must be mapped to database user accounts in order to work with database objects.</span></span> <span data-ttu-id="e0397-127">データベース ユーザーをデータベース ロールに追加すると、そのロールに関連付けられたすべての権限セットが継承されます。</span><span class="sxs-lookup"><span data-stu-id="e0397-127">Database users can then be added to database roles, inheriting any permission sets associated with those roles.</span></span> <span data-ttu-id="e0397-128">すべての権限を付与できます。</span><span class="sxs-lookup"><span data-stu-id="e0397-128">All permissions can be granted.</span></span>  
  
 <span data-ttu-id="e0397-129">アプリケーションのセキュリティを計画する際は、`public` ロール、`dbo` ユーザー アカウント、および `guest` アカウントも考慮する必要があります。</span><span class="sxs-lookup"><span data-stu-id="e0397-129">You must also consider the `public` role, the `dbo` user account, and the `guest` account when you design security for your application.</span></span>  
  
### <a name="the-public-role"></a><span data-ttu-id="e0397-130">public ロール</span><span class="sxs-lookup"><span data-stu-id="e0397-130">The public Role</span></span>  
 <span data-ttu-id="e0397-131">`public` ロールは、システム データベースを含め、すべてのデータベースに存在します。</span><span class="sxs-lookup"><span data-stu-id="e0397-131">The `public` role is contained in every database, which includes system databases.</span></span> <span data-ttu-id="e0397-132">このロールは削除できません。また、ユーザーを追加したり削除したりすることもできません。</span><span class="sxs-lookup"><span data-stu-id="e0397-132">It cannot be dropped and you cannot add or remove users from it.</span></span> <span data-ttu-id="e0397-133">他のすべてのユーザーおよびロールは既定で `public` ロールに所属するため、`public` ロールに付与された権限を継承します。</span><span class="sxs-lookup"><span data-stu-id="e0397-133">Permissions granted to the `public` role are inherited by all other users and roles because they belong to the `public` role by default.</span></span> <span data-ttu-id="e0397-134">`public` には、すべてのユーザーに割り当てる必要のある権限だけを付与してください。</span><span class="sxs-lookup"><span data-stu-id="e0397-134">Grant `public` only the permissions you want all users to have.</span></span>  
  
### <a name="the-dbo-user-account"></a><span data-ttu-id="e0397-135">dbo ユーザー アカウント</span><span class="sxs-lookup"><span data-stu-id="e0397-135">The dbo User Account</span></span>  
 <span data-ttu-id="e0397-136">`dbo` (データベース所有者) は、データベースのすべてのアクティビティを実行する暗黙権限を持ったユーザー アカウントです。</span><span class="sxs-lookup"><span data-stu-id="e0397-136">The `dbo`, or database owner, is a user account that has implied permissions to perform all activities in the database.</span></span> <span data-ttu-id="e0397-137">`sysadmin` 固定サーバー ロールのメンバーは、自動的に `dbo` にマップされます。</span><span class="sxs-lookup"><span data-stu-id="e0397-137">Members of the `sysadmin` fixed server role are automatically mapped to `dbo`.</span></span>  
  
> [!NOTE]
>  `dbo` <span data-ttu-id="e0397-138">説明したように、スキーマの名前をも[所有権と SQL Server のユーザーとスキーマの分離](../../../../../docs/framework/data/adonet/sql/ownership-and-user-schema-separation-in-sql-server.md)します。</span><span class="sxs-lookup"><span data-stu-id="e0397-138">is also the name of a schema, as discussed in [Ownership and User-Schema Separation in SQL Server](../../../../../docs/framework/data/adonet/sql/ownership-and-user-schema-separation-in-sql-server.md).</span></span>  
  
 <span data-ttu-id="e0397-139">`dbo` ユーザー アカウントは、よく `db_owner` 固定データベース ロールと混同されます。</span><span class="sxs-lookup"><span data-stu-id="e0397-139">The `dbo` user account is frequently confused with the `db_owner` fixed database role.</span></span> <span data-ttu-id="e0397-140">`db_owner` のスコープがサーバー全体であるのに対し、`sysadmin` のスコープはデータベースです。</span><span class="sxs-lookup"><span data-stu-id="e0397-140">The scope of `db_owner` is a database; the scope of `sysadmin` is the whole server.</span></span> <span data-ttu-id="e0397-141">`db_owner` ロールのメンバーであるからといって、`dbo` のユーザー権限があるとは限りません。</span><span class="sxs-lookup"><span data-stu-id="e0397-141">Membership in the `db_owner` role does not confer `dbo` user privileges.</span></span>  
  
### <a name="the-guest-user-account"></a><span data-ttu-id="e0397-142">guest ユーザー アカウント</span><span class="sxs-lookup"><span data-stu-id="e0397-142">The guest User Account</span></span>  
 <span data-ttu-id="e0397-143">認証されて SQL Server のインスタンスへのログインが許可されたユーザーが、その後、データベースにアクセスするためには各データベースに別個のユーザー アカウントが存在している必要があります。</span><span class="sxs-lookup"><span data-stu-id="e0397-143">After a user has been authenticated and allowed to log in to an instance of SQL Server, a separate user account must exist in each database the user has to access.</span></span> <span data-ttu-id="e0397-144">データベースごとにユーザー アカウントが要求されるため、SQL Server のインスタンスに接続したとしても、ユーザーは、サーバー上のすべてのデータベースにアクセスできるわけではありません。</span><span class="sxs-lookup"><span data-stu-id="e0397-144">Requiring a user account in each database prevents users from connecting to an instance of SQL Server and accessing all the databases on a server.</span></span> <span data-ttu-id="e0397-145">この要件を回避するには、データベースに `guest` ユーザー アカウントを設定します。guest ユーザーは、データベースにユーザー アカウントがなくてもデータベースにログインしてアクセスできます。</span><span class="sxs-lookup"><span data-stu-id="e0397-145">The existence of a `guest` user account in the database circumvents this requirement by allowing a login without a database user account to access a database.</span></span>  
  
 <span data-ttu-id="e0397-146">`guest` アカウントは、SQL Server のすべてのバージョンが備えている組み込みのアカウントです。</span><span class="sxs-lookup"><span data-stu-id="e0397-146">The `guest` account is a built-in account in all versions of SQL Server.</span></span> <span data-ttu-id="e0397-147">新しいデータベースでは、既定で無効化されています。</span><span class="sxs-lookup"><span data-stu-id="e0397-147">By default, it is disabled in new databases.</span></span> <span data-ttu-id="e0397-148">このアカウントを有効にした場合でも、Transact-SQL の REVOKE CONNECT FROM GUEST ステートメントを実行して、CONNECT 権限を取り消すことによって無効にできます。</span><span class="sxs-lookup"><span data-stu-id="e0397-148">If it is enabled, you can disable it by revoking its CONNECT permission by executing the Transact-SQL REVOKE CONNECT FROM GUEST statement.</span></span>  
  
> [!IMPORTANT]
>  <span data-ttu-id="e0397-149">`guest` アカウントはできるだけ使用しないようにしてください。ユーザーごとに割り当てられたデータベース権限がなくても、すべてのログイン ユーザーが、このアカウントに付与されているデータベース権限を取得できるようになります。</span><span class="sxs-lookup"><span data-stu-id="e0397-149">Avoid using the `guest` account; all logins without their own database permissions obtain the database permissions granted to this account.</span></span> <span data-ttu-id="e0397-150">`guest` アカウントを使用する必要がある場合は、最小限の権限を付与するようにしてください。</span><span class="sxs-lookup"><span data-stu-id="e0397-150">If you must use the `guest` account, grant it minimum permissions.</span></span>
