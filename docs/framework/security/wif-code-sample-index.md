# WIF コード サンプル インデックス
Windows アイデンティティ基盤の 4.5 のコード サンプルを次に示します。  
  
-   [ClaimsAwareWebApp](http://go.microsoft.com/fwlink/?LinkID=248405) \-この認証 externalization の基本的な使い方 （、ローカルをテストするセキュリティ トークン サービスの Id およびアクセス ツールから Visual Studio 11\) は、従来の ASP のサンプルします。NET アプリケーション \(としてではなく、web サイトへ\)。  
  
-   [ClaimsAwareWebService](http://go.microsoft.com/fwlink/?LinkID=248406) 、このサンプルでは、従来の WCF サービスでの認証 externalization の基本的な使用します。  
  
-   [ClaimsAwareMvcApplication](http://go.microsoft.com/fwlink/?LinkID=248407) \- このサンプルたんだぜを一括ではない保護などの MVC に統合する方法を示していて、フォーム認証を使用するコード ログオンのコント ローラーをリダイレクトします。  
  
-   [ClaimsAwareWebFarm](http://go.microsoft.com/fwlink/?LinkID=248408) \-大きなクッキーを交換するのではなく参照によってセッションを使用することができますようにこのサンプル ファーム準備がセッション キャッシュには、tokenreplycache\) を示します。  また、ファーム内の cookie をセキュリティ保護する簡単な方法を示します。  
  
-   [ClaimsAwareFormsAuthentication](http://go.microsoft.com/fwlink/?LinkID=248409)ので、この非常に単純な例を示します。NET 4.5 がユーザーを認証する方法に関係なく、プリンシパルにクレームを取得します。  
  
-   [ClaimsBasedAuthorization](http://go.microsoft.com/fwlink/?LinkID=248410)\- このサンプルは、独自の承認ポリシーを適用するため、CLaimsAuthorizationManager のクラスと、ClaimsAuthorizationModule を使用する方法を示しています。  
  
-   [FederationMetadata](http://go.microsoft.com/fwlink/?LinkID=248411) ： このサンプル \(カスタム STS\) での動的な生成と動的消費 \(には、依存元のサードパーティ製アプリケーション\) のメタデータ ドキュメントの両方を示しています。  
  
-   [CustomToken](http://go.microsoft.com/fwlink/?LinkID=248412) ： このサンプル簡易 Web トークン \(SWT） のカスタム トークン型を構築する方法を示しています。
