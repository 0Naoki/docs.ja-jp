# IPv6 のルーティング
柔軟な処理方法は IPv6 の福利厚生です。  IPv4 のネットワーク ID があり、割り当てられた方法により、大規模な工順の表に、インターネット バックボーンにあるルーターが管理している必要があります。  これらのルーターは、インターネットでの各ノードにべきではインデックスをパケットのようにすべての工順を知る必要があります。  住所を集計する機能と IPv6 は、フレックス住所を使用すると、包括的に工順テーブルのサイズが減少します。  この新しい住所アーキテクチャで、中間ルーターは、メッセージを適切に設定します。にネットワーク内の保管場所の部分のみの追跡を保たなければ必要があります。  
  
## 横に検出  
 横の検出によって提供される機能は、次のとおりです。:  
  
-   Router の検出。  これはホストが保管場所のルーターを識別することができます。  
  
-   住所の解決。  これはノードが対応するホップ次の住所 \(決済 ARP プロトコル\[住所\]のリンク レイヤの住所の交換決済されるようにします\)。  
  
-   住所の自動設定。  これはホストが自動的にサービス拠点ローカルおよびグローバル アドレスをコンフィギュレーションすることができます。  
  
 横に検出は、次のような IPv6 \(ICMPv6\) メッセージに対して制御メッセージ インターネット プロトコルを使用します:  
  
-   Router 広告。  時定期的にまたはルーター願いに応じてのルーターに送信。  住所の状態、接頭辞、およびそのほかのパラメータを提供する IPv6 のルーターを使用ルーター広告。  
  
-   Router 願い。  リンクのルーターがルーター広告をすぐにするように要求するにホストで送信する。  
  
-   横願い。  、重複する住所の検出住所の決済のノードに送信、または隣人がまだ到達可能であることを確認します。  
  
-   横の広告。  横願いに応答するか、またはリンク レイヤの住所の変更の隣人を通知するためにノードに送信。  
  
-   リダイレクト。  出荷ノードの特定のデスティネーションでは、次ホップの住所を表示するにルーターに送信。
