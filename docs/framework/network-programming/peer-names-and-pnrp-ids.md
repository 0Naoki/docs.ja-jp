# ピア名と PNRP ID
ピアの名前は、コンピュータ、ユーザー グループ、サービス、または IPv6 の住所に決済できるピアに関連付けられているあらゆるものになる場合がある通信のエンドポイントを表します。  ピア決済 Protocol \(PNRP\) 名はクラウドのメンバを識別するために使用される PNRP ID の作成の統計的に固有ピアの名前を取得します。  
  
## ピアの名前  
 ピアの名前は、それらのないとして登録されているまたは保証することができます。  保証の名前はないユーザーが重複しない保証の名前を登録できるため、からかうになる、文字列で。  保証のない名前は、プライベートまたは別の方法で保護されたネットワークで保持されます。  保護された名前は証明書およびデジタル署名と保護されます。  元の発行元のみ保護された名前の所有権を証明できます。  
  
 クラウドおよび範囲の組み合わせが PNRP 活動を入力するピアに妥当に安全な環境を提供します。  ただし、保護されたピアの名前を使用してネットワーキング アプリケーションの全体的なセキュリティはありません。  アプリケーションのセキュリティは実装に依存します。  
  
 保護されたピアの名前が所有者だけが登録され、公開キーとインターネット保護されます。  ピアの保護された名前に対応するデータ キーを持つピア エンティティによって所有していたと見なされます。  会社データは、コンフィギュレーション キーを使用して署名する認定ピアの住所 \(CPA\) が証明することができます。  悪意のあるユーザーが対応するデータなしでピア キーの名前の所有権を作成することはできません。  
  
## PNRP ID  
 ![PNRP ID](../../../docs/framework/network-programming/media/fdc9e8a0-4a1c-488d-a019-bc3a1973220c.png "fdc9e8a0\-4a1c\-488d\-a019\-bc3a1973220c")  
  
 PNRP ID が次に構成されます:  
  
-   ピアツーピア \(P2P\) ID として知られる高位 128 ビットのコードは、エンドポイントに割り当てられたピアの名前のハッシュです。  ピアの名前は、次の形式があります: *Authority.Classifier*.  保護された名前に、*権限は* 16 桁数な文字 \(ピアの名前の公開キーの安全なアルゴリズム ハッシュ 1 \(SHA1 ハッシュ\) です。  保証のない名前に、*権限は* 一つの文字「0 "です。  *分類子。* アプリケーションを識別する文字列で。  ピアの名前の分類子。`null` のターミネーターを含む 149 文字以下で超えることはできません。  
  
-   最低位 128 ビットのコードが同じクラウドの同じ P2P ID の異なるインスタンスを識別する生成された番号であるサービスの場所に使用されます。  
  
 P2P ID およびサービスの場所のこの組み合わせは複数の PNRP ID が単一のコンピュータから登録されます。
