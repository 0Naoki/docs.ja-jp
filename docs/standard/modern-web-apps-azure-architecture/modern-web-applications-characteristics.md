## <a name="modular-and-loosely-coupled"></a><span data-ttu-id="59449-131">モジュール式と疎結合</span><span class="sxs-lookup"><span data-stu-id="59449-131">Modular and Loosely Coupled</span></span>

## <a name="easily-tested-with-automated-tests"></a><span data-ttu-id="59449-137">自動テストで簡単にテスト</span><span class="sxs-lookup"><span data-stu-id="59449-137">Easily Tested with Automated Tests</span></span>

## <a name="traditional-and-spa-behaviors-supported"></a><span data-ttu-id="59449-142">サポートされている従来の動作と SPA の動作</span><span class="sxs-lookup"><span data-stu-id="59449-142">Traditional and SPA Behaviors Supported</span></span>

<span data-ttu-id="59449-143">従来の Web アプリケーションでは、クライアント側の動作はほとんどありませんでしたが、代わりに、アプリケーションに必要な可能性があるすべてのナビゲーション、クエリ、および更新にはサーバーに依存していました。</span><span class="sxs-lookup"><span data-stu-id="59449-143">Traditional web applications have involved little client-side behavior, but instead have relied on the server for all navigation, queries, and updates the app might need to make.</span></span> <span data-ttu-id="59449-144">ユーザーが実行した個々の新しい操作は新しい Web 要求に変換されるので、結果としてエンド ユーザーのブラウザーでページ全体の再読み込みが実行されます。</span><span class="sxs-lookup"><span data-stu-id="59449-144">Each new operation made by the user would be translated into a new web request, with the result being a full page reload in the end user's browser.</span></span> <span data-ttu-id="59449-145">一般的に、従来のモデル ビュー コントローラー (MVC) フレームワークはこのアプローチに従い、個々の新しい要求は異なるコントローラー アクションに対応しているので、結果的にモデルと連携し、ビューを返します。</span><span class="sxs-lookup"><span data-stu-id="59449-145">Classic Model-View-Controller (MVC) frameworks typically follow this approach, with each new request corresponding to a different controller action, which in turn would work with a model and return a view.</span></span> <span data-ttu-id="59449-146">AJAX (Asynchronous JavaScript and XML) 機能を使用すると、特定のページ上の個々の操作が強化される場合がありますが、アプリケーションの全体的なアーキテクチャではさまざまな MVC ビューと URL エンドポイントが使用されていました。</span><span class="sxs-lookup"><span data-stu-id="59449-146">Some individual operations on a given page might be enhanced with AJAX (Asynchronous JavaScript and XML) functionality, but the overall architecture of the app used many different MVC views and URL endpoints.</span></span>

<span data-ttu-id="59449-147">これに対し、シングル ページ アプリケーション (SPA) の場合は、動的に生成されるサーバー側のページ読み込み (存在する場合) はほとんどありません。</span><span class="sxs-lookup"><span data-stu-id="59449-147">Single Page Applications (SPAs), by contrast, involve very few dynamically generated server-side page loads (if any).</span></span> <span data-ttu-id="59449-148">多くの SPA は、アプリケーションを起動して実行するために必要な JavaScript ライブラリを読み込む静的 HTML ファイル内で初期化されます。</span><span class="sxs-lookup"><span data-stu-id="59449-148">Many SPAs are initialized within a static HTML file which loads the necessary JavaScript libraries to start and run the app.</span></span> <span data-ttu-id="59449-149">このようなアプリケーションはデータのニーズに合わせて Web API を多用し、より高度なユーザー エクスペリエンスを提供できます。</span><span class="sxs-lookup"><span data-stu-id="59449-149">These apps make heavy usage of web APIs for their data needs, and can provide much richer user experiences.</span></span>

<span data-ttu-id="59449-150">多くの Web アプリケーションでは、従来の Web アプリケーションの動作 (通常はコンテンツ用) と SPA (対話機能用) を組み合わせています。</span><span class="sxs-lookup"><span data-stu-id="59449-150">Many web applications involve a combination of traditional web application behavior (typically for content) and SPAs (for interactivity).</span></span> <span data-ttu-id="59449-151">ASP.NET Core は、同じアプリケーション内で同じツール セットと基礎のフレームワーク ライブラリを使用して MVC と Web API の両方をサポートしています。</span><span class="sxs-lookup"><span data-stu-id="59449-151">ASP.NET Core supports both MVC and web APIs in the same application, using the same set of tools and underlying framework libraries.</span></span>
