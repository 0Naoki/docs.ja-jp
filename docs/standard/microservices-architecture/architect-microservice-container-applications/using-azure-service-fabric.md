## <a name="stateless-versus-stateful-microservices"></a><span data-ttu-id="91d80-150">ステートレス マイクロサービスとステートフル マイクロサービスの比較</span><span class="sxs-lookup"><span data-stu-id="91d80-150">Stateless versus stateful microservices</span></span>

<span data-ttu-id="91d80-151">前述のように、各マイクロサービス (論理的な境界指定されたコンテキスト) は、ドメイン モデル (データとロジック) を所有している必要があります。</span><span class="sxs-lookup"><span data-stu-id="91d80-151">As mentioned earlier, each microservice (logical Bounded Context) must own its domain model (data and logic).</span></span> <span data-ttu-id="91d80-152">ステートレス マイクロ サービスの場合、データベースはサービス外部に用意することになります。このデータベースは、SQL Server などのリレーショナル データベースでも、MongoDB、Azure Cosmos DB などの NoSQL データベースでもかまいません。</span><span class="sxs-lookup"><span data-stu-id="91d80-152">In the case of stateless microservices, the databases will be external, employing relational options like SQL Server, or NoSQL options like MongoDB or Azure Cosmos DB.</span></span>

<span data-ttu-id="91d80-153">Service Fabric でサービス自体もステートフルにして、データをマイクロサービス内に常駐させることもできます。</span><span class="sxs-lookup"><span data-stu-id="91d80-153">But the services themselves can also be stateful in Service Fabric, which means that the data resides within the microservice.</span></span> <span data-ttu-id="91d80-154">このデータは、同じサーバー上に存在できるだけでなく、マイクロサービスのプロセス内であれば、メモリ内に存在することも、ハード ドライブに保存することも、他のノードにレプリケートすることもできます。</span><span class="sxs-lookup"><span data-stu-id="91d80-154">This data might exist not just on the same server, but within the microservice process, in memory and persisted on hard drives and replicated to other nodes.</span></span> <span data-ttu-id="91d80-155">図 4-30 では、別の方法を示します。</span><span class="sxs-lookup"><span data-stu-id="91d80-155">Figure 4-30 shows the different approaches.</span></span>

<span data-ttu-id="91d80-158">ステートレス アプローチは、完全に有効で、従来の既知のパターンに似ているので、ステートフルなマイクロサービスよりも容易に実装できます。</span><span class="sxs-lookup"><span data-stu-id="91d80-158">A stateless approach is perfectly valid and is easier to implement than stateful microservices, since the approach is similar to traditional and well-known patterns.</span></span> <span data-ttu-id="91d80-159">ステートレス マイクロサービスは、プロセスおよびデータ ソース間の遅延を強制します。</span><span class="sxs-lookup"><span data-stu-id="91d80-159">But stateless microservices impose latency between the process and data sources.</span></span> <span data-ttu-id="91d80-160">追加のキャッシュとキューを使用してパフォーマンスを改善しようとしている場合は、含まれる移動する部分が増加します。</span><span class="sxs-lookup"><span data-stu-id="91d80-160">They also involve more moving pieces when you are trying to improve performance with additional cache and queues.</span></span> <span data-ttu-id="91d80-161">結果として、階層の数が多すぎる複雑なアーキテクチャになる可能性があります。</span><span class="sxs-lookup"><span data-stu-id="91d80-161">The result is that you can end up with complex architectures that have too many tiers.</span></span>
