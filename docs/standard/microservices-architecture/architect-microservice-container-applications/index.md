## <a name="container-design-principles"></a><span data-ttu-id="cd205-112">コンテナー設計の原則</span><span class="sxs-lookup"><span data-stu-id="cd205-112">Container design principles</span></span>

<span data-ttu-id="cd205-113">コンテナー モデルでは、コンテナー イメージのインスタンスは 1 つのプロセスを表します。</span><span class="sxs-lookup"><span data-stu-id="cd205-113">In the container model, a container image instance represents a single process.</span></span> <span data-ttu-id="cd205-114">コンテナー イメージのプロセス境界として定義すると、プロセスの拡張やバッチ処理に使用できるプリミティブを作成できます。</span><span class="sxs-lookup"><span data-stu-id="cd205-114">By defining a container image as a process boundary, you can create primitives that can be used to scale the process or to batch it.</span></span>

<span data-ttu-id="cd205-115">コンテナー イメージをデザインするときには、[ENTRYPOINT](https://docs.docker.com/engine/reference/builder/) 定義が Dockerfile に表示されます。</span><span class="sxs-lookup"><span data-stu-id="cd205-115">When you design a container image, you will see an [ENTRYPOINT](https://docs.docker.com/engine/reference/builder/) definition in the Dockerfile.</span></span> <span data-ttu-id="cd205-116">これで定義するプロセスの有効期間が、コンテナーの有効期間を制御します。</span><span class="sxs-lookup"><span data-stu-id="cd205-116">This defines the process whose lifetime controls the lifetime of the container.</span></span> <span data-ttu-id="cd205-117">プロセスが完了すると、コンテナーのライフサイクルが終了します。</span><span class="sxs-lookup"><span data-stu-id="cd205-117">When the process completes, the container lifecycle ends.</span></span> <span data-ttu-id="cd205-118">コンテナーは、Web サーバーなどの実行時間の長いプロセスを表す場合がありますが、以前は Azure の [WebJobs](https://docs.microsoft.com/azure/app-service-web/websites-webjobs-resources) として実装される場合があったバッチ ジョブなどの有効期間が短いプロセスも表すことができます。</span><span class="sxs-lookup"><span data-stu-id="cd205-118">Containers might represent long-running processes like web servers, but can also represent short-lived processes like batch jobs, which formerly might have been implemented as Azure [WebJobs](https://docs.microsoft.com/azure/app-service-web/websites-webjobs-resources).</span></span>

<span data-ttu-id="cd205-119">プロセスが失敗した場合、コンテナーが終了し、Orchestrator が引き継ぎます。</span><span class="sxs-lookup"><span data-stu-id="cd205-119">If the process fails, the container ends, and the orchestrator takes over.</span></span> <span data-ttu-id="cd205-120">Orchestrator が 5 つのインスタンスの実行を維持するように構成され、1 つが失敗した場合、Orchestrator は、失敗したプロセスを置換する別のコンテナー インスタンスを作成します。</span><span class="sxs-lookup"><span data-stu-id="cd205-120">If the orchestrator was configured to keep five instances running and one fails, the orchestrator will create another container instance to replace the failed process.</span></span> <span data-ttu-id="cd205-121">バッチ ジョブで、プロセスはパラメーターを指定して開始されます。</span><span class="sxs-lookup"><span data-stu-id="cd205-121">In a batch job, the process is started with parameters.</span></span> <span data-ttu-id="cd205-122">プロセスが完了すると、作業が完了します。</span><span class="sxs-lookup"><span data-stu-id="cd205-122">When the process completes, the work is complete.</span></span> <span data-ttu-id="cd205-123">このガイダンスでは、後でオーケストレーターについてドリル ダウンします。</span><span class="sxs-lookup"><span data-stu-id="cd205-123">This guidance drills-down on orchestrators, later on.</span></span>

<span data-ttu-id="cd205-124">1 つのコンテナーで複数のプロセスが実行されているシナリオが見られる場合があります。</span><span class="sxs-lookup"><span data-stu-id="cd205-124">You might find a scenario where you want multiple processes running in a single container.</span></span> <span data-ttu-id="cd205-125">このシナリオでは、コンテナーごとに 1 つのエントリ ポイントのみが存在できるため、必要な数のプログラムを起動するスクリプトをコンテナー内で実行できます。</span><span class="sxs-lookup"><span data-stu-id="cd205-125">For that scenario, since there can be only one entry point per container, you could run a script within the container that launches as many programs as needed.</span></span> <span data-ttu-id="cd205-126">たとえば、[スーパーバイザー](http://supervisord.org/)または同様のツールを使用して、1 つのコンテナー内で複数のプロセスの起動を処理できます。</span><span class="sxs-lookup"><span data-stu-id="cd205-126">For example, you can use [Supervisor](http://supervisord.org/) or a similar tool to take care of launching multiple processes inside a single container.</span></span> <span data-ttu-id="cd205-127">ただし、コンテナーごとに複数のプロセスを保持するアーキテクチャが見られる場合でも、その方法はあまり一般的ではありません。</span><span class="sxs-lookup"><span data-stu-id="cd205-127">However, even though you can find architectures that hold multiple processes per container, that approach is not very common.</span></span>
