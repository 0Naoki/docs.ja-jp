 <span data-ttu-id="5f394-167">パターンは単純です。</span><span class="sxs-lookup"><span data-stu-id="5f394-167">The pattern is very simple.</span></span> <span data-ttu-id="5f394-168">必要な作業は、<xref:System.Runtime.Serialization.ISerializable> インターフェイスを実装し、オブジェクトを逆シリアル化するときに使用する特別なコンストラクターを指定するだけです。</span><span class="sxs-lookup"><span data-stu-id="5f394-168">All you need to do is implement the <xref:System.Runtime.Serialization.ISerializable> interface and provide a special constructor that is used when the object is deserialized.</span></span>  
  
 <span data-ttu-id="5f394-169">**✓ DO** シリアル化コンス トラクターを保護して、型指定された、という名前のサンプルは、ここに示すとおりに正確に 2 つのパラメーターを提供します。</span><span class="sxs-lookup"><span data-stu-id="5f394-169">**✓ DO** make the serialization constructor protected and provide two parameters typed and named exactly as shown in the sample here.</span></span>

```  
public class Person : ISerializable {  
    protected Person(SerializationInfo info, StreamingContext context) {  
        ...  
    }  
}  
```  
  
 <span data-ttu-id="5f394-170">**✓ DO** 実装、`ISerializable`メンバー明示的にします。</span><span class="sxs-lookup"><span data-stu-id="5f394-170">**✓ DO** implement the `ISerializable` members explicitly.</span></span>  
  
 <span data-ttu-id="5f394-171">**✓ DO** にリンク確認要求を適用<xref:System.Runtime.Serialization.ISerializable.GetObjectData%2A?displayProperty=nameWithType>実装します。</span><span class="sxs-lookup"><span data-stu-id="5f394-171">**✓ DO** apply a link demand to <xref:System.Runtime.Serialization.ISerializable.GetObjectData%2A?displayProperty=nameWithType> implementation.</span></span> <span data-ttu-id="5f394-172">これにより、コアとランタイム シリアライザーは、メンバーにアクセス権を持つのみ完全に信頼できます。</span><span class="sxs-lookup"><span data-stu-id="5f394-172">This ensures that only fully trusted core and the Runtime Serializer have access to the member.</span></span>
