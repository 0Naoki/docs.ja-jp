## <a name="naming-conventions"></a><span data-ttu-id="afd7d-116">名前付け規則</span><span class="sxs-lookup"><span data-stu-id="afd7d-116">Naming Conventions</span></span>  
 <span data-ttu-id="afd7d-117">.NET Framework の型では、階層構造を伴うドット構文の名前付け方法が使われます。</span><span class="sxs-lookup"><span data-stu-id="afd7d-117">.NET Framework types use a dot syntax naming scheme that connotes a hierarchy.</span></span> <span data-ttu-id="afd7d-118">この方法では、関連する型が名前空間にグループ化されるため、検索と参照を簡単に行うことができます。</span><span class="sxs-lookup"><span data-stu-id="afd7d-118">This technique groups related types into namespaces so they can be searched and referenced more easily.</span></span> <span data-ttu-id="afd7d-119">フルネームのうち右端のドットまでの最初の部分は、名前空間の名前です。</span><span class="sxs-lookup"><span data-stu-id="afd7d-119">The first part of the full name — up to the rightmost dot — is the namespace name.</span></span> <span data-ttu-id="afd7d-120">名前の最後の部分は型名です。</span><span class="sxs-lookup"><span data-stu-id="afd7d-120">The last part of the name is the type name.</span></span> <span data-ttu-id="afd7d-121">たとえば、**System.Collections.ArrayList** は **ArrayList** 型であり、名前空間 **System.Collections** に属します。</span><span class="sxs-lookup"><span data-stu-id="afd7d-121">For example, **System.Collections.ArrayList** represents the **ArrayList** type, which belongs to the **System.Collections** namespace.</span></span> <span data-ttu-id="afd7d-122">**System.Collections** 内の型を使用することで、オブジェクトのコレクションを操作できます。</span><span class="sxs-lookup"><span data-stu-id="afd7d-122">The types in **System.Collections** can be used to manipulate collections of objects.</span></span>  
  
 <span data-ttu-id="afd7d-123">この名前付け方法によって、[!INCLUDE[dnprdnshort](../../includes/dnprdnshort-md.md)] を拡張するライブラリ開発者は、型の階層構造のグループを簡単に作成し、一貫性のあるわかりやすい名前を付けることができます。</span><span class="sxs-lookup"><span data-stu-id="afd7d-123">This naming scheme makes it easy for library developers extending the [!INCLUDE[dnprdnshort](../../includes/dnprdnshort-md.md)] to create hierarchical groups of types and name them in a consistent, informative manner.</span></span> <span data-ttu-id="afd7d-124">また、型の完全名 (つまり名前空間と型名) によって型を明確に特定できるため、型名の競合を防ぐことができます。</span><span class="sxs-lookup"><span data-stu-id="afd7d-124">It also allows types to be unambiguously identified by their full name (that is, by their namespace and type name), which prevents type name collisions.</span></span> <span data-ttu-id="afd7d-125">ライブラリ開発者は、次の規則に従って名前空間の名前を付けてください。</span><span class="sxs-lookup"><span data-stu-id="afd7d-125">Library developers are expected to use the following convention when creating names for their namespaces:</span></span>  
  
 <span data-ttu-id="afd7d-126">*CompanyName*.*TechnologyName*</span><span class="sxs-lookup"><span data-stu-id="afd7d-126">*CompanyName*.*TechnologyName*</span></span>  
  
 <span data-ttu-id="afd7d-127">たとえば、名前空間 Microsoft.Word はこのガイドラインに従っています。</span><span class="sxs-lookup"><span data-stu-id="afd7d-127">For example, the namespace Microsoft.Word conforms to this guideline.</span></span>  
  
 <span data-ttu-id="afd7d-128">名前付けパターンを使用して関連する型を名前空間にグループ化する方法は、クラス ライブラリを構築および文書化するのに便利です。</span><span class="sxs-lookup"><span data-stu-id="afd7d-128">The use of naming patterns to group related types into namespaces is a very useful way to build and document class libraries.</span></span> <span data-ttu-id="afd7d-129">ただし、この名前付け方法は、参照可能範囲、メンバー アクセス、継承、セキュリティ、バインディングには影響しません。</span><span class="sxs-lookup"><span data-stu-id="afd7d-129">However, this naming scheme has no effect on visibility, member access, inheritance, security, or binding.</span></span> <span data-ttu-id="afd7d-130">名前空間は複数のアセンブリにまたがって分割でき、また 1 つのアセンブリに複数の名前空間からの型を含めることができます。</span><span class="sxs-lookup"><span data-stu-id="afd7d-130">A namespace can be partitioned across multiple assemblies and a single assembly can contain types from multiple namespaces.</span></span> <span data-ttu-id="afd7d-131">アセンブリは、共通言語ランタイムにおけるバージョン管理、配置、セキュリティ、読み込み、および参照可能範囲のための構造を提供します。</span><span class="sxs-lookup"><span data-stu-id="afd7d-131">The assembly provides the formal structure for versioning, deployment, security, loading, and visibility in the common language runtime.</span></span>
