## <a name="stateless-versus-stateful-microservices"></a><span data-ttu-id="001ae-239">ステートレス マイクロサービスとステートフル マイクロサービスの比較</span><span class="sxs-lookup"><span data-stu-id="001ae-239">Stateless versus stateful microservices</span></span>

<span data-ttu-id="001ae-240">前述のように、各マイクロサービス (論理的な境界指定されたコンテキスト) は、ドメイン モデル (データとロジック) を所有している必要があります。</span><span class="sxs-lookup"><span data-stu-id="001ae-240">As mentioned earlier, each microservice (logical Bounded Context) must own its domain model (data and logic).</span></span> <span data-ttu-id="001ae-241">場合は、ステートレス マイクロ サービス データベースを SQL Server のようなリレーショナル オプションまたは MongoDB、Azure DocumentDB などの NoSQL オプションを使用して、外部になります。</span><span class="sxs-lookup"><span data-stu-id="001ae-241">In the case of stateless microservices, the databases will be external, employing relational options like SQL Server, or NoSQL options like MongoDB or Azure DocumentDB.</span></span>

<span data-ttu-id="001ae-242">サービス自体もできるステートフルなマイクロ サービス内のデータが置かれているためです。</span><span class="sxs-lookup"><span data-stu-id="001ae-242">But the services themselves also can be stateful, which means that the data resides within the microservice.</span></span> <span data-ttu-id="001ae-243">このデータは、だけでなく、同じサーバーでは、メモリ内のマイクロ サービス プロセス内に存在可能性がありますとドライブに保存され、その他のノードにレプリケートされます。</span><span class="sxs-lookup"><span data-stu-id="001ae-243">This data might exist not just on the same server, but within the microservice process, in memory, and persisted on drives and replicated to other nodes.</span></span> <span data-ttu-id="001ae-244">図 4-13 は、いくつかの方法を示しています。</span><span class="sxs-lookup"><span data-stu-id="001ae-244">Figure 4-13 shows the different approaches.</span></span>

<span data-ttu-id="001ae-246">ステートレスな方法は、完全に有効な従来から使い慣れているパターンに似ているために、ステートフル マイクロ サービスよりも実装する方が簡単です。</span><span class="sxs-lookup"><span data-stu-id="001ae-246">A stateless approach is perfectly valid and is easier to implement than stateful microservices because the approach is similar to traditional and well-known patterns.</span></span> <span data-ttu-id="001ae-247">ステートレス マイクロサービスは、プロセスおよびデータ ソース間の遅延を強制します。</span><span class="sxs-lookup"><span data-stu-id="001ae-247">But stateless microservices impose latency between the process and data sources.</span></span> <span data-ttu-id="001ae-248">追加のキャッシュとキューを使用してパフォーマンスを改善しようとしている場合は、含まれる移動する部分が増加します。</span><span class="sxs-lookup"><span data-stu-id="001ae-248">They also involve more moving pieces when you are trying to improve performance with additional cache and queues.</span></span> <span data-ttu-id="001ae-249">結果として、階層の数が多すぎる複雑なアーキテクチャになる可能性があります。</span><span class="sxs-lookup"><span data-stu-id="001ae-249">The result is that you can end up with complex architectures that have too many tiers.</span></span>
