# <a name="differences-between-passing-an-argument-by-value-and-by-reference-visual-basic"></a><span data-ttu-id="5f72f-102">引数の値渡しと参照渡しの違い (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="5f72f-102">Differences Between Passing an Argument By Value and By Reference (Visual Basic)</span></span>
<span data-ttu-id="5f72f-103">プロシージャに 1 つまたは複数の引数を渡すと、ときに、各引数は、呼び出し元のコード内の基になるプログラミング要素に対応します。</span><span class="sxs-lookup"><span data-stu-id="5f72f-103">When you pass one or more arguments to a procedure, each argument corresponds to an underlying programming element in the calling code.</span></span> <span data-ttu-id="5f72f-104">この基になる要素の値またはへの参照を渡すことができます。</span><span class="sxs-lookup"><span data-stu-id="5f72f-104">You can pass either the value of this underlying element, or a reference to it.</span></span> <span data-ttu-id="5f72f-105">これと呼ばれますが、*渡し*。</span><span class="sxs-lookup"><span data-stu-id="5f72f-105">This is known as the *passing mechanism*.</span></span>  
  
## <a name="passing-by-value"></a><span data-ttu-id="5f72f-106">値渡し</span><span class="sxs-lookup"><span data-stu-id="5f72f-106">Passing by Value</span></span>  
 <span data-ttu-id="5f72f-107">引数を渡す*値によって*指定することによって、 [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md)プロシージャの定義に対応するパラメーターのキーワード。</span><span class="sxs-lookup"><span data-stu-id="5f72f-107">You pass an argument *by value* by specifying the [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="5f72f-108">この渡すメカニズムを使用して、Visual Basic は、プロシージャ内のローカル変数に基になるプログラミング要素の値をコピーします。</span><span class="sxs-lookup"><span data-stu-id="5f72f-108">When you use this passing mechanism, Visual Basic copies the value of the underlying programming element into a local variable in the procedure.</span></span> <span data-ttu-id="5f72f-109">プロシージャのコードには、呼び出し元のコードで、基になる要素へのアクセスがありません。</span><span class="sxs-lookup"><span data-stu-id="5f72f-109">The procedure code does not have any access to the underlying element in the calling code.</span></span>  
  
## <a name="passing-by-reference"></a><span data-ttu-id="5f72f-110">参照渡しで渡す</span><span class="sxs-lookup"><span data-stu-id="5f72f-110">Passing by Reference</span></span>  
 <span data-ttu-id="5f72f-111">引数を渡す*参照によって*指定することによって、 [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md)プロシージャの定義に対応するパラメーターのキーワード。</span><span class="sxs-lookup"><span data-stu-id="5f72f-111">You pass an argument *by reference* by specifying the [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="5f72f-112">この渡すメカニズムを使用して、Visual Basic により、プロシージャ基になるプログラミング要素への直接参照では呼び出し元のコード。</span><span class="sxs-lookup"><span data-stu-id="5f72f-112">When you use this passing mechanism, Visual Basic gives the procedure a direct reference to the underlying programming element in the calling code.</span></span>  
  
## <a name="passing-mechanism-and-element-type"></a><span data-ttu-id="5f72f-113">引き渡し方法および要素の型</span><span class="sxs-lookup"><span data-stu-id="5f72f-113">Passing Mechanism and Element Type</span></span>  
 <span data-ttu-id="5f72f-114">渡す機能の選択は、基になる要素型の分類と同じではありません。</span><span class="sxs-lookup"><span data-stu-id="5f72f-114">The choice of passing mechanism is not the same as the classification of the underlying element type.</span></span> <span data-ttu-id="5f72f-115">値渡しまたは参照によって渡すことは、プロシージャのコードを提供する Visual Basic を指します。</span><span class="sxs-lookup"><span data-stu-id="5f72f-115">Passing by value or by reference refers to what Visual Basic supplies to the procedure code.</span></span> <span data-ttu-id="5f72f-116">値型または参照型は、メモリ内プログラミング要素を格納する方法を参照します。</span><span class="sxs-lookup"><span data-stu-id="5f72f-116">A value type or reference type refers to how a programming element is stored in memory.</span></span>  
  
 <span data-ttu-id="5f72f-117">ただし、引き渡し方法および要素の型は相互にします。</span><span class="sxs-lookup"><span data-stu-id="5f72f-117">However, the passing mechanism and element type are interrelated.</span></span> <span data-ttu-id="5f72f-118">参照型の値は、メモリ内の他の場所でのデータへのポインターです。</span><span class="sxs-lookup"><span data-stu-id="5f72f-118">The value of a reference type is a pointer to the data elsewhere in memory.</span></span> <span data-ttu-id="5f72f-119">つまり、基になる要素自体にアクセスできない場合でも値渡しで参照型を渡すときにプロシージャ コードが、基になる要素のデータへのポインターを持っています。</span><span class="sxs-lookup"><span data-stu-id="5f72f-119">This means that when you pass a reference type by value, the procedure code has a pointer to the underlying element's data, even though it cannot access the underlying element itself.</span></span> <span data-ttu-id="5f72f-120">たとえば、要素が、配列変数の場合は、プロシージャのコードでは、変数自体にアクセスできないが、配列メンバーにアクセスできます。</span><span class="sxs-lookup"><span data-stu-id="5f72f-120">For example, if the element is an array variable, the procedure code does not have access to the variable itself, but it can access the array members.</span></span>
