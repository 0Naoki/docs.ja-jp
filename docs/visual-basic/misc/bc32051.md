# <a name="type-argument-inference-failed-for-type-parameter-typeparametername-of-genericproceduresignature"></a><span data-ttu-id="faa5a-102">入力引数を推定の型パラメーターに失敗しました '\<typeparametername >' の'\<genericproceduresignature >'</span><span class="sxs-lookup"><span data-stu-id="faa5a-102">Type argument inference failed for type parameter '\<typeparametername>' of '\<genericproceduresignature>'</span></span>
<span data-ttu-id="faa5a-103">入力引数を推定の型パラメーターに失敗しました '\<typeparametername >' の'\<genericproceduresignature >'。</span><span class="sxs-lookup"><span data-stu-id="faa5a-103">Type argument inference failed for type parameter '\<typeparametername>' of '\<genericproceduresignature>'.</span></span> <span data-ttu-id="faa5a-104">型引数をパラメーターに渡される引数から推論できませんでした '\<parametername >'。</span><span class="sxs-lookup"><span data-stu-id="faa5a-104">Type argument could not be inferred from the argument passed to parameter '\<parametername>'.</span></span>  
  
 <span data-ttu-id="faa5a-105">ジェネリック プロシージャが型引数を指定せずに呼び出されたため、コンパイラはパラメーターのいずれかに渡す型を推定できません。</span><span class="sxs-lookup"><span data-stu-id="faa5a-105">A generic procedure is called without supplying any type arguments, and the compiler cannot infer the type to pass to one of the parameters.</span></span>  
  
 <span data-ttu-id="faa5a-106">通常、ジェネリック プロシージャを呼び出す場合は、ジェネリック プロシージャで定義する型パラメーターごとに型引数を指定します。</span><span class="sxs-lookup"><span data-stu-id="faa5a-106">Normally, when you call a generic procedure, you supply a type argument for each type parameter that the generic procedure defines.</span></span> <span data-ttu-id="faa5a-107">型引数を指定しない場合、コンパイラは型パラメーターに渡す型を推定しようとします。</span><span class="sxs-lookup"><span data-stu-id="faa5a-107">If you do not supply any type arguments, then the compiler attempts to infer the types to be passed to the type parameters.</span></span> <span data-ttu-id="faa5a-108">呼び出しのコンテキストから型パラメーターについて競合するデータ型情報が提供される場合、型の推定は失敗します。</span><span class="sxs-lookup"><span data-stu-id="faa5a-108">If the context of the call provides conflicting data type information for a type parameter, then the type inference fails.</span></span>  
  
 <span data-ttu-id="faa5a-109">次のコードでは、このエラーが生成される可能性があります。</span><span class="sxs-lookup"><span data-stu-id="faa5a-109">The following code can generate this error.</span></span>  
  
```  
Public Sub doSomething(Of t)(ByVal arg1 As t(), ByVal arg2 As t)  
End Sub  
Call doSomething(6, 42)  
```  
  
 <span data-ttu-id="faa5a-110">前の例では、コンパイラは `Integer` に渡される 42 という値に基づいて `t` に対する型 `arg2`を推定します。</span><span class="sxs-lookup"><span data-stu-id="faa5a-110">In the preceding example, the compiler infers type `Integer` for `t` based on the value of 42 passed to `arg2`.</span></span> <span data-ttu-id="faa5a-111">ただし、その推定には `arg1` が型 `Integer()`(つまり、 `Integer`の配列) である必要があり、 `arg1` に渡される値 6 はその型と一致しません。</span><span class="sxs-lookup"><span data-stu-id="faa5a-111">However, that inference would require `arg1` to be of type `Integer()`, that is, an array of `Integer`, and the value 6 passed to `arg1` does not match that type.</span></span>  
  
 <span data-ttu-id="faa5a-112">**エラー ID:** BC32051</span><span class="sxs-lookup"><span data-stu-id="faa5a-112">**Error ID:** BC32051</span></span>
