# <a name="net-core-distribution-packaging"></a><span data-ttu-id="6bf50-103">.NET Core の配布パッケージ</span><span class="sxs-lookup"><span data-stu-id="6bf50-103">.NET Core distribution packaging</span></span>

<span data-ttu-id="6bf50-104">.NET Core はますます多くのプラットフォームで使用できるようになってきているため、.NET Core をパッケージ化し、名前を付け、バージョン管理する方法を知っていると便利です。</span><span class="sxs-lookup"><span data-stu-id="6bf50-104">As .NET Core becomes available on more and more platforms, it's useful to learn how to package, name, and version it.</span></span> <span data-ttu-id="6bf50-105">そうすることで、パッケージの管理者は、ユーザーの .NET の実行環境に左右されることなく一貫した体験が保証されるようにサポートできます。</span><span class="sxs-lookup"><span data-stu-id="6bf50-105">This way, package maintainers can help ensure a consistent experience no matter where users choose to run .NET.</span></span>

## <a name="disk-layout"></a><span data-ttu-id="6bf50-106">ディスク レイアウト</span><span class="sxs-lookup"><span data-stu-id="6bf50-106">Disk layout</span></span>

<span data-ttu-id="6bf50-107">インストールした .NET Core は複数のコンポーネントで構成されています。コンポーネントは、ファイルシステムで次のようにレイアウトされています。</span><span class="sxs-lookup"><span data-stu-id="6bf50-107">When installed, .NET Core consists of several components that are layed out as follows in the filesystem:</span></span>

```
.
├── dotnet                       (1)
├── LICENSE.txt                  (8)
├── ThirdPartyNotices.txt        (8)
├── host
│   └── fxr
│       └── <fxr version>        (2)
├── sdk
│   ├── <sdk version>            (3)
│   └── NuGetFallbackFolder      (4)
└── shared
    ├── Microsoft.NETCore.App
    │   └── <runtime version>    (5)
    └── Microsoft.AspNetCore.App
        └── <aspnetcore version> (6)
    └── Microsoft.AspNetCore.All
        └── <aspnetcore version> (7)
/
├─usr/share/man/man1
│       └── dotnet.1.gz          (9)
└─usr/bin
        └── dotnet               (10)
```

- <span data-ttu-id="6bf50-108">(1) **dotnet** ホスト (別名 "muxer (マルチプレクサー)") には 2 つの異なるロールがあります。アプリケーションを起動するランタイムのアクティブ化と、コマンドをディスパッチする SDK のアクティブ化です。</span><span class="sxs-lookup"><span data-stu-id="6bf50-108">(1) **dotnet** The host (also known as the "muxer") has two distinct roles: activate a runtime to launch an application, and activate an SDK to dispatch commands to it.</span></span> <span data-ttu-id="6bf50-109">ホストはネイティブの実行可能ファイルです (`dotnet.exe`)。</span><span class="sxs-lookup"><span data-stu-id="6bf50-109">The host is a native executable (`dotnet.exe`).</span></span>

<span data-ttu-id="6bf50-110">ホストは 1 つですが、他のコンポーネントのほとんどはバージョン管理されたディレクトリに入っています (2、3、5、6)。</span><span class="sxs-lookup"><span data-stu-id="6bf50-110">While there is a single host, most of the other components are in versioned directories (2,3,5,6).</span></span> <span data-ttu-id="6bf50-111">並列インストールされており、複数のバージョンをシステムに置くことができます。</span><span class="sxs-lookup"><span data-stu-id="6bf50-111">These means multiple versions can be present on the system since they are installed side-by-side.</span></span>

- <span data-ttu-id="6bf50-112">(2) **host/fxr/\<fxr バージョン>** には、ホストが使用するフレームワーク解決ロジックが含まれます。</span><span class="sxs-lookup"><span data-stu-id="6bf50-112">(2) **host/fxr/\<fxr version>** contains the framework resolution logic used by the host.</span></span> <span data-ttu-id="6bf50-113">ホストでは、インストールされている最新の hostfxr が使用されます。</span><span class="sxs-lookup"><span data-stu-id="6bf50-113">The host uses the latest hostfxr that is installed.</span></span> <span data-ttu-id="6bf50-114">hostfxr は、.NET Core アプリケーションの実行時に適切なランタイムを選択する役割を担います。</span><span class="sxs-lookup"><span data-stu-id="6bf50-114">The hostfxr is responsible for selecting the appropriate runtime when executing a .NET Core application.</span></span> <span data-ttu-id="6bf50-115">たとえば、.NET Core 2.0.0 用としてビルドされたアプリケーションは、2.0.5 が利用できればそれを利用します。</span><span class="sxs-lookup"><span data-stu-id="6bf50-115">For example, an application built for .NET Core 2.0.0 will use the 2.0.5 runtime when it is available.</span></span> <span data-ttu-id="6bf50-116">同様に、hostfxr は開発中、適切な SDK を選択します。</span><span class="sxs-lookup"><span data-stu-id="6bf50-116">Similarly, hostfxr selects the appropriate SDK during development.</span></span>

- <span data-ttu-id="6bf50-117">(3) **sdk/\<sdk バージョン>** SDK (別名 "ツール") は、.NET Core のライブラリやアプリケーションを記述し、ビルドするために使用できるマネージ ツールのセットです。</span><span class="sxs-lookup"><span data-stu-id="6bf50-117">(3) **sdk/\<sdk version>** The SDK (also known as "the tooling") is a set of managed tools that can be used to write and build .NET Core libraries and applications.</span></span> <span data-ttu-id="6bf50-118">SDK には、CLI、Roslyn コンパイラ、MSBuild、関連するビルド タスクとターゲット、NuGet、新しいプロジェクト テンプレートなどが含まれています。</span><span class="sxs-lookup"><span data-stu-id="6bf50-118">The SDK includes the CLI, the Roslyn compiler, MSBuild, and associated build tasks and targets, NuGet, new project templates, etc.</span></span>

- <span data-ttu-id="6bf50-119">(4) **sdk/NuGetFallbackFolder** には、`dotnet restore` 手順中、SDK で使用される NuGet パッケージのキャッシュが含まれています。</span><span class="sxs-lookup"><span data-stu-id="6bf50-119">(4) **sdk/NuGetFallbackFolder** contains a cache of NuGet packages used by an SDK during the `dotnet restore` step.</span></span>

<span data-ttu-id="6bf50-120">**共有**フォルダーには、フレームワークが含まれています。</span><span class="sxs-lookup"><span data-stu-id="6bf50-120">The **shared** folder contains frameworks.</span></span> <span data-ttu-id="6bf50-121">共有フレームワークは、さまざまなアプリケーションで利用できるように、中央の場所で一連のライブラリを提供します。</span><span class="sxs-lookup"><span data-stu-id="6bf50-121">A shared framework provides a set of libraries at a central location so they can be used by different applications.</span></span>

- <span data-ttu-id="6bf50-122">(5) **shared/Microsoft.NETCore.App/\<runtime バージョン>** このフレームワークには、.NET Core のランタイムと補助マネージ ライブラリが含まれています。</span><span class="sxs-lookup"><span data-stu-id="6bf50-122">(5) **shared/Microsoft.NETCore.App/\<runtime version>** This framework contains the .NET Core runtime and supporting managed libraries.</span></span>

- <span data-ttu-id="6bf50-123">(6,7) **shared/Microsoft.AspNetCore.{App,All}/\<aspnetcore バージョン>** には、ASP.NET Core ライブラリが含まれています。</span><span class="sxs-lookup"><span data-stu-id="6bf50-123">(6,7) **shared/Microsoft.AspNetCore.{App,All}/\<aspnetcore version>** contains the ASP.NET Core libraries.</span></span> <span data-ttu-id="6bf50-124">`Microsoft.AspNetCore.App` の下にあるライブラリは、.NET Core プロジェクトの一部として開発され、サポートされています。</span><span class="sxs-lookup"><span data-stu-id="6bf50-124">The libraries under `Microsoft.AspNetCore.App` are developed and supported as part of the .NET Core project.</span></span> <span data-ttu-id="6bf50-125">`Microsoft.AspNetCore.All` の下にあるライブラリは、サードパーティ ライブラリも含まれるスーパーセットです。</span><span class="sxs-lookup"><span data-stu-id="6bf50-125">The libraries under `Microsoft.AspNetCore.All` are a superset which also contains 3rd party libraries.</span></span>

- <span data-ttu-id="6bf50-126">(8) **LICENSE.txt,ThirdPartyNotices.txt** は .NET Core ライセンスと、.NET Core で利用されるサードパーティ ライブラリのライセンスです。</span><span class="sxs-lookup"><span data-stu-id="6bf50-126">(8) **LICENSE.txt,ThirdPartyNotices.txt** are the .NET Core license and licenses of third-party libraries used in .NET Core.</span></span>

- <span data-ttu-id="6bf50-127">(9,10) **dotnet.1.gz, dotnet** `dotnet.1.gz` は dotnet man ページです。</span><span class="sxs-lookup"><span data-stu-id="6bf50-127">(9,10) **dotnet.1.gz, dotnet** `dotnet.1.gz` is the dotnet man page.</span></span> <span data-ttu-id="6bf50-128">`dotnet` は dotnet host(1) のシンボリック リンクです。</span><span class="sxs-lookup"><span data-stu-id="6bf50-128">`dotnet` is a symlink to the dotnet host(1).</span></span> <span data-ttu-id="6bf50-129">これらのファイルは、システム統合のために、よく知られている場所にインストールされます。</span><span class="sxs-lookup"><span data-stu-id="6bf50-129">These files are installed at well known locations for system integration.</span></span>

## <a name="recommended-packages"></a><span data-ttu-id="6bf50-130">推奨パッケージ</span><span class="sxs-lookup"><span data-stu-id="6bf50-130">Recommended packages</span></span>

<span data-ttu-id="6bf50-131">.NET Core バージョン管理は、ランタイム コンポーネント `[major].[minor]` バージョン番号に基づきます。</span><span class="sxs-lookup"><span data-stu-id="6bf50-131">.NET Core versioning is based on the runtime component `[major].[minor]` version numbers.</span></span>
<span data-ttu-id="6bf50-132">SDK バージョンは同じ `[major].[minor]` を利用し、SDK の機能とパッチ意味論を結合する非依存の `[patch]` が与えられます。</span><span class="sxs-lookup"><span data-stu-id="6bf50-132">The SDK version uses the same `[major].[minor]` and has an independent `[patch]` which combines feature and patch semantics for the SDK.</span></span>
<span data-ttu-id="6bf50-133">例: SDK バージョン 2.2.302 は、2.2 ランタイム対応の SDK の第 3 機能リリースの第 2 パッチ リリースです。</span><span class="sxs-lookup"><span data-stu-id="6bf50-133">For example: SDK version 2.2.302 is a the 2nd patch release of the 3rd feature release of the SDK that supports the 2.2 runtime.</span></span>

<span data-ttu-id="6bf50-134">パッケージには、その名前にバージョン番号の一部が含まれているものもあります。</span><span class="sxs-lookup"><span data-stu-id="6bf50-134">Some of the packages include part of the version number in their name.</span></span> <span data-ttu-id="6bf50-135">それによって、エンドユーザーは特定のバージョンをインストールできます。</span><span class="sxs-lookup"><span data-stu-id="6bf50-135">This allows the end-user to install a specific version.</span></span>
