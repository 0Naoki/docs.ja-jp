# <a name="net-core-version-history"></a><span data-ttu-id="cd97a-103">.NET Core のバージョン履歴</span><span class="sxs-lookup"><span data-stu-id="cd97a-103">NET Core version history</span></span>

<span data-ttu-id="cd97a-104">.NET Core のバージョン番号は、.NET Core SDK と .NET Core ランタイムが異なる頻度でリリースされているため、わかりにくくなっています。</span><span class="sxs-lookup"><span data-stu-id="cd97a-104">Version numbers for .NET Core are challenging because .NET Core SDK and .NET Core Runtime release on different cadences.</span></span> <span data-ttu-id="cd97a-105">頻度が異なっているのは、チームが次の 3 つのうちの 2 のみを行う必要があったことを意味します。</span><span class="sxs-lookup"><span data-stu-id="cd97a-105">The different cadences mean the team was forced to do only two of the following three things:</span></span>

1. <span data-ttu-id="cd97a-106">特にツール、C# および VB を .NET Core ランタイムよりも早く進められるように、単独でリリースする。</span><span class="sxs-lookup"><span data-stu-id="cd97a-106">Release independently, specifically allowing tools, C# and VB to advance faster than the .NET Core Runtime.</span></span>
2. <span data-ttu-id="cd97a-107">.NET Core SDK と .NET Core ランタイムのバージョン アラインメントを維持する。</span><span class="sxs-lookup"><span data-stu-id="cd97a-107">Maintain alignment in version numbers between .NET Core SDK and .NET Core Runtime.</span></span>
3. <span data-ttu-id="cd97a-108">.NET Core SDK と .NET Core ランタイムの両方にセマンティック バージョニングを使用する。</span><span class="sxs-lookup"><span data-stu-id="cd97a-108">Use semantic versioning for both the .NET Core SDK and .NET Core Runtime.</span></span>

<span data-ttu-id="cd97a-109">2.0.0 では、バージョン アラインメントを実行し、1 つのリリースを円滑に進めました。</span><span class="sxs-lookup"><span data-stu-id="cd97a-109">2.0.0 forced version alignment and proceeded smoothly for one release.</span></span> <span data-ttu-id="cd97a-110">2017 年 12 月、.NET Core SDK では機能リリースがありましたが、.NET Core ランタイムでは対応するリリースはありませんでした。</span><span class="sxs-lookup"><span data-stu-id="cd97a-110">In December 2017 .NET Core SDK had a feature release, with no corresponding release in the .NET Core Runtime.</span></span> <span data-ttu-id="cd97a-111">チームは、目標の 1 と 3 を選択し、.NET Core ランタイムと SDK 間のアラインメントは失われることになりました。</span><span class="sxs-lookup"><span data-stu-id="cd97a-111">The team chose goals 1 and 3, losing alignment between the .NET Core Runtime and SDK.</span></span> <span data-ttu-id="cd97a-112">.NET Core ランタイム 2.1 がリリースされるまでに、いくつかの .NET Core SDK 2.1.x バージョンがリリースされました。</span><span class="sxs-lookup"><span data-stu-id="cd97a-112">Several .NET Core SDK 2.1.x versions were released before .NET Core Runtime 2.1.</span></span> <span data-ttu-id="cd97a-113">SDK には上位互換性がないため、SDK 2.1.x のこれらのバージョンは、.NET Core ランタイム 2.1 を対象にできませんでした。</span><span class="sxs-lookup"><span data-stu-id="cd97a-113">Since the SDK is not forwards compatible, these 2.1.x SDK versions could not target .NET Core Runtime 2.1.</span></span> <span data-ttu-id="cd97a-114">チームは、「[.NET Core バージョン管理](index.md#versioning-details)」に記載されているように、セマンティック バージョニングを放棄して、目標 1 と 2 に切り替えることによって、大きな混乱に対応しました。</span><span class="sxs-lookup"><span data-stu-id="cd97a-114">The team responded to the considerable confusion by switching to goals 1 and 2, abandoning semantic versioning as described in [.NET Core versioning](index.md#versioning-details).</span></span>

<span data-ttu-id="cd97a-115">セマンティック バージョニングの破棄が決定されるまでのタイミングのため、バージョン番号 2.1.10x から 2.1.20x までの間に暫定リリースがありましたが、こちらも .NET Core ランタイム 2.1 を対象とすることはできません。</span><span class="sxs-lookup"><span data-stu-id="cd97a-115">Because of the timing of the decision to abandon semantic versioning, there were transitional releases in the 2.1.10x and 2.1.20x version number ranges that also can't target .NET Core Runtime 2.1.</span></span>

<span data-ttu-id="cd97a-116">バージョン番号の最初の 2 桁が、.NET Core ランタイムの 2.1.0 バージョンと、NET Core SDK の 2.1.300 バージョンと再度アラインされます。</span><span class="sxs-lookup"><span data-stu-id="cd97a-116">The first two digits of the version numbers realign with the 2.1.0 version of the .NET Core Runtime and the 2.1.300 version of the .NET Core SDK.</span></span>
