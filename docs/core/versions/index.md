## <a name="semantic-versioning"></a><span data-ttu-id="97cb4-146">セマンティック バージョン管理</span><span class="sxs-lookup"><span data-stu-id="97cb4-146">Semantic versioning</span></span>

<span data-ttu-id="97cb4-147">.NET Core *ランタイム*は、[セマンティック バージョニング (SemVer)](https://semver.org/) にほぼ準拠しています。`MAJOR.MINOR.PATCH` バージョン管理の使用が採用され、バージョン番号のさまざまな部分を使用して変更の程度と種類が記述されています。</span><span class="sxs-lookup"><span data-stu-id="97cb4-147">The .NET Core *Runtime* roughly adheres to [Semantic Versioning (SemVer)](https://semver.org/), adopting the use of `MAJOR.MINOR.PATCH` versioning, using the various parts of the version number to describe the degree and type of change.</span></span>

## <a name="relationship-to-net-standard-versions"></a><span data-ttu-id="97cb4-178">.NET Standard との関係</span><span class="sxs-lookup"><span data-stu-id="97cb4-178">Relationship to .NET Standard versions</span></span>

<span data-ttu-id="97cb4-179">.NET Standard は、.NET 参照アセンブリで構成されています。</span><span class="sxs-lookup"><span data-stu-id="97cb4-179">.NET Standard consists of a .NET reference assembly.</span></span> <span data-ttu-id="97cb4-180">各プラットフォームに固有の複数の実装があります。</span><span class="sxs-lookup"><span data-stu-id="97cb4-180">There are multiple implementations specific to each platform.</span></span> <span data-ttu-id="97cb4-181">参照アセンブリには、指定された .NET Standard バージョンの一部である .NET API の定義が含まれています。</span><span class="sxs-lookup"><span data-stu-id="97cb4-181">The reference assembly contains the definition of .NET APIs which are part of a given .NET Standard version.</span></span> <span data-ttu-id="97cb4-182">各実装では、特定のプラットフォームに対する .NET Standard コントラクトが満たされます。</span><span class="sxs-lookup"><span data-stu-id="97cb4-182">Each implementation fulfills the .NET Standard contract on the specific platform.</span></span> <span data-ttu-id="97cb4-183">.NET Standard の詳細については、.NET ガイド内の [.NET Standard](../../standard/net-standard.md) に関する記事を参照してください。</span><span class="sxs-lookup"><span data-stu-id="97cb4-183">You can learn more about .NET Standard in the article on [.NET Standard](../../standard/net-standard.md) in the .NET Guide.</span></span>

<span data-ttu-id="97cb4-184">.NET Standard 参照アセンブリでは、`MAJOR.MINOR` バージョン管理スキームが使用されます。</span><span class="sxs-lookup"><span data-stu-id="97cb4-184">The .NET Standard reference assembly uses a `MAJOR.MINOR` versioning scheme.</span></span> <span data-ttu-id="97cb4-185">.NET Standard の場合、`PATCH` レベルは有用ではありません。.NET Standard では、API 仕様しか公開されず (実装は対象外)、定義上、API に対する変更はいずれも機能セット内の変更を示すものであり、`MINOR` バージョンとなるためです。</span><span class="sxs-lookup"><span data-stu-id="97cb4-185">`PATCH` level isn't useful for .NET Standard because it exposes only an API specification (no implementation) and by definition any change to the API would represent a change in the feature set, and thus a new `MINOR` version.</span></span>

<span data-ttu-id="97cb4-186">各プラットフォーム上の実装の場合は、通常、プラットフォーム リリースの一部として更新されるので、そのプラットフォーム上で .NET Standard を使用しているプログラマには明らかにされません。</span><span class="sxs-lookup"><span data-stu-id="97cb4-186">The implementations on each platform may be updated, typically as part of the platform release, and thus not evident to the programmers using .NET Standard on that platform.</span></span>
