# <a name="compiler-error-cs0172"></a><span data-ttu-id="f2da0-102">コンパイラ エラー CS0172</span><span class="sxs-lookup"><span data-stu-id="f2da0-102">Compiler Error CS0172</span></span>
<span data-ttu-id="f2da0-103">'type1' と 'type2' が暗黙的に変換し合うため、条件式の型がわかりません</span><span class="sxs-lookup"><span data-stu-id="f2da0-103">Type of conditional expression cannot be determined because 'type1' and 'type2' implicitly convert to one another</span></span>  
  
<span data-ttu-id="f2da0-104">条件付きステートメントでする必要がありますいずれかの側の型を暗黙的に変換する、`:`トークンです。</span><span class="sxs-lookup"><span data-stu-id="f2da0-104">In a conditional statement, you must be able to implicitly convert the types on either side of the `:` token.</span></span> <span data-ttu-id="f2da0-105">また、することはできません。 相互の暗黙的な変換1 つの変換をだけです。</span><span class="sxs-lookup"><span data-stu-id="f2da0-105">Also, there cannot be mutual implicit conversions; you only need one conversion.</span></span>
  
<span data-ttu-id="f2da0-106">次の例では CS0172 が生成されます。</span><span class="sxs-lookup"><span data-stu-id="f2da0-106">The following sample generates CS0172:</span></span>
  
```csharp  
// CS0172.cs  
public class Square  
{  
   public class Circle  
   {  
      public static implicit operator Circle(Square aa)  
      {  
         return null;  
      }  
  
      public static implicit operator Square(Circle aa)  
      // using explicit resolves this error  
      // public static explicit operator Square(Circle aa)  
      {  
         return null;  
      }  
   }  
  
   public static void Main()  
   {  
      Circle aa = new Circle();  
      Square ii = new Square();  
      object o = (1 == 1) ? aa : ii;   // CS0172  
      // the following cast would resolve this error  
      // (1 == 1) ? aa : (Circle)ii;  
   }  
}  
```
