<span data-ttu-id="34b96-103">次の例では、メソッドに[構造体](../../../csharp/language-reference/keywords/struct.md)を渡すことと[クラス](../../../csharp/language-reference/keywords/class.md) インスタンスを渡すことの違いを示します。</span><span class="sxs-lookup"><span data-stu-id="34b96-103">The following example demonstrates how passing a [struct](../../../csharp/language-reference/keywords/struct.md) to a method differs from passing a [class](../../../csharp/language-reference/keywords/class.md) instance to a method.</span></span> <span data-ttu-id="34b96-104">この例では、両方の引数 (構造体とクラス インスタンス) が値によって渡され、両方のメソッドが引数の 1 つのフィールドの値を変更します。</span><span class="sxs-lookup"><span data-stu-id="34b96-104">In the example, both of the arguments (struct and class instance) are passed by value, and both methods change the value of one field of the argument.</span></span> <span data-ttu-id="34b96-105">ただし、2 つのメソッドの結果は同じではありません。構造体を渡した場合に渡される内容と、クラスのインスタンスを渡した場合に渡される内容が異なるためです。</span><span class="sxs-lookup"><span data-stu-id="34b96-105">However, the results of the two methods are not the same because what is passed when you pass a struct differs from what is passed when you pass an instance of a class.</span></span>  
  
 <span data-ttu-id="34b96-106">構造体は[値型](../../../csharp/language-reference/keywords/value-types.md)であるため、メソッドに[構造体が値によって渡される](../../../csharp/programming-guide/classes-and-structs/passing-value-type-parameters.md)と、メソッドは構造体引数のコピーを受け取って操作します。</span><span class="sxs-lookup"><span data-stu-id="34b96-106">Because a struct is a [value type](../../../csharp/language-reference/keywords/value-types.md), when you [pass a struct by value](../../../csharp/programming-guide/classes-and-structs/passing-value-type-parameters.md) to a method, the method receives and operates on a copy of the struct argument.</span></span> <span data-ttu-id="34b96-107">メソッドは、呼び出し側メソッドの元の構造体にはアクセスできないため、どのような場合でもこの構造体を変更することはできません。</span><span class="sxs-lookup"><span data-stu-id="34b96-107">The method has no access to the original struct in the calling method and therefore can't change it in any way.</span></span> <span data-ttu-id="34b96-108">メソッドで変更できるのはコピーのみです。</span><span class="sxs-lookup"><span data-stu-id="34b96-108">The method can change only the copy.</span></span>  
  
 <span data-ttu-id="34b96-109">クラス インスタンスは、値型ではなく、[参照型](../../../csharp/language-reference/keywords/reference-types.md)です。</span><span class="sxs-lookup"><span data-stu-id="34b96-109">A class instance is a [reference type](../../../csharp/language-reference/keywords/reference-types.md), not a value type.</span></span> <span data-ttu-id="34b96-110">メソッドに[参照型が値によって渡される](../../../csharp/programming-guide/classes-and-structs/passing-reference-type-parameters.md)と、メソッドはクラス インスタンスへの参照のコピーを受け取ります。</span><span class="sxs-lookup"><span data-stu-id="34b96-110">When [a reference type is passed by value](../../../csharp/programming-guide/classes-and-structs/passing-reference-type-parameters.md) to a method, the method receives a copy of the reference to the class instance.</span></span> <span data-ttu-id="34b96-111">つまり、メソッドは、インスタンス自体のコピーではなく、インスタンスのアドレスのコピーを受け取ることになります。</span><span class="sxs-lookup"><span data-stu-id="34b96-111">That is, the method receives a copy of the address of the instance, not a copy of the instance itself.</span></span> <span data-ttu-id="34b96-112">呼び出し側メソッドのクラス インスタンスにはアドレスがあり、呼び出されたメソッドのパラメータにはそのアドレスのコピーがあり、両方のアドレスが同じオブジェクトを参照します。</span><span class="sxs-lookup"><span data-stu-id="34b96-112">The class instance in the calling method has an address, the parameter in the called method has a copy of the address, and both addresses refer to the same object.</span></span> <span data-ttu-id="34b96-113">パラメーターにはアドレスのコピーのみが含まれるため、呼び出されたメソッドは呼び出し側メソッドのクラス インスタンスのアドレスを変更できません。</span><span class="sxs-lookup"><span data-stu-id="34b96-113">Because the parameter contains only a copy of the address, the called method cannot change the address of the class instance in the calling method.</span></span> <span data-ttu-id="34b96-114">ただし、呼び出されたメソッドはアドレスを使用して、元のアドレスとコピーの両方が参照するクラス メンバーにアクセスできます。</span><span class="sxs-lookup"><span data-stu-id="34b96-114">However, the called method can use the address to access the class members that both the original address and the copy reference.</span></span> <span data-ttu-id="34b96-115">呼び出されたメソッドがクラス メンバーを変更すると、呼び出し側メソッドの元のクラス インスタンスも変更されます。</span><span class="sxs-lookup"><span data-stu-id="34b96-115">If the called method changes a class member, the original class instance in the calling method also changes.</span></span>  
  
 <span data-ttu-id="34b96-116">次の例の出力はこの違いを示しています。</span><span class="sxs-lookup"><span data-stu-id="34b96-116">The output of the following example illustrates the difference.</span></span> <span data-ttu-id="34b96-117">クラス インスタンスの `willIChange` フィールドの値はメソッド `ClassTaker` の呼び出しによって変更されます。これは、メソッドがパラメーターのアドレスを使用して、クラス インスタンスの指定されたフィールドを検索するためです。</span><span class="sxs-lookup"><span data-stu-id="34b96-117">The value of the `willIChange` field of the class instance is changed by the call to method `ClassTaker` because the method uses the address in the parameter to find the specified field of the class instance.</span></span> <span data-ttu-id="34b96-118">呼び出し側メソッドの構造体の `willIChange` フィールドはメソッド `StructTaker` の呼び出しによって変更されません。これは、引数の値が、そのアドレスのコピーではなく、構造体自体のコピーであるためです。</span><span class="sxs-lookup"><span data-stu-id="34b96-118">The `willIChange` field of the struct in the calling method is not changed by the call to method `StructTaker` because the value of the argument is a copy of the struct itself, not a copy of its address.</span></span> <span data-ttu-id="34b96-119">`StructTaker` はコピーを変更し、そのコピーは、`StructTaker` の呼び出しが完了したときに失われます。</span><span class="sxs-lookup"><span data-stu-id="34b96-119">`StructTaker` changes the copy, and the copy is lost when the call to `StructTaker` is completed.</span></span>
