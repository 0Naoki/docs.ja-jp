# <a name="compiler-error-cs1009"></a><span data-ttu-id="4ed7a-102">コンパイラ エラー CS1009</span><span class="sxs-lookup"><span data-stu-id="4ed7a-102">Compiler Error CS1009</span></span>
<span data-ttu-id="4ed7a-103">認識できないエスケープ シーケンスです</span><span class="sxs-lookup"><span data-stu-id="4ed7a-103">Unrecognized escape sequence</span></span>  
  
 <span data-ttu-id="4ed7a-104">予期しない文字は [文字列](../../../csharp/language-reference/keywords/string.md)の円記号 (\\) に従います。</span><span class="sxs-lookup"><span data-stu-id="4ed7a-104">An unexpected character follows a backslash (\\) in a [string](../../../csharp/language-reference/keywords/string.md).</span></span> <span data-ttu-id="4ed7a-105">コンパイラは、有効なエスケープ文字のうち 1 つを要求します。</span><span class="sxs-lookup"><span data-stu-id="4ed7a-105">The compiler expects one of the valid escape characters.</span></span> <span data-ttu-id="4ed7a-106">詳細については、「[文字のエスケープ](../../../standard/base-types/character-escapes-in-regular-expressions.md)」を参照してください。</span><span class="sxs-lookup"><span data-stu-id="4ed7a-106">For more information, see [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span></span>  
  
 <span data-ttu-id="4ed7a-107">次の例では CS1009 が生成されます。</span><span class="sxs-lookup"><span data-stu-id="4ed7a-107">The following sample generates CS1009.</span></span>  
  
```csharp  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";     
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 <span data-ttu-id="4ed7a-108">このエラーの一般的な原因は、次の例のように、ファイル名に円記号が使用されていることです。</span><span class="sxs-lookup"><span data-stu-id="4ed7a-108">A common cause of this error is using the backslash character in a file name, as the following example shows.</span></span>  
  
```csharp  
string filename = "c:\myFolder\myFile.txt";  
```  
  
 <span data-ttu-id="4ed7a-109">このエラーを解決するには、次の例のように「\\\\」または @-quoted のリテラル文字列を使用します。</span><span class="sxs-lookup"><span data-stu-id="4ed7a-109">To resolve this error, use "\\\\" or the @-quoted string literal, as the following example shows.</span></span>  
  
```csharp  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";     
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```
