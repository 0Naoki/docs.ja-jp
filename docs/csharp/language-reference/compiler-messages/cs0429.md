# <a name="compiler-warning-level-4-cs0429"></a><span data-ttu-id="47367-102">コンパイラの警告 (レベル 4) CS0429</span><span class="sxs-lookup"><span data-stu-id="47367-102">Compiler Warning (level 4) CS0429</span></span>
<span data-ttu-id="47367-103">到達できない式コードが検出されました</span><span class="sxs-lookup"><span data-stu-id="47367-103">Unreachable expression code detected</span></span>  
  
 <span data-ttu-id="47367-104">このエラーは、コード内の式の一部にアクセスできないときに発生します。</span><span class="sxs-lookup"><span data-stu-id="47367-104">This error occurs whenever part of an expression in your code is unreachable.</span></span> <span data-ttu-id="47367-105">`&&` 演算の左辺が常に false であり、`myTest()` メソッドは評価されないため、次の例の条件 `false && myTest()` はこの基準を満たしています。</span><span class="sxs-lookup"><span data-stu-id="47367-105">In the following example, the condition `false && myTest()` meets this criteria because the `myTest()` method will never get evaluated due to the fact that the left side of the `&&` operation is always false.</span></span> <span data-ttu-id="47367-106">`&&` 演算子で `false` ステートメントが false と評価されるとすぐに評価は停止され、右辺は評価されません。</span><span class="sxs-lookup"><span data-stu-id="47367-106">As soon as the `&&` operator evaluates the `false` statement as false, it stops the evaluation, and will never evaluate the right side.</span></span>

```csharp  
{  
    public static void Main()   
    {  
        if (false && myTest())  // CS0429  
        // Try the following line instead:  
        // if (true && myTest())  
        {  
        }  
        else  
        {  
            int i = 0;  
            i++;  
        }  
    }  
  
    static bool myTest() { return true; }  
}  
```
