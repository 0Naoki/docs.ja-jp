# Compiler Error CS0201
[!INCLUDE[vs2017banner](../../../csharp/includes/vs2017banner.md)]

割り当て、呼び出し、インクリメント、デクリメント、および新しいオブジェクトの式のみがステートメントとして使用できます。  
  
 無効なステートメントが検出されると、コンパイラでこのエラーが生成されます。  無効なステートメントとは、セミコロンで終わる任意の行または一連の行のうち、代入 \([\=](../../../csharp/language-reference/operators/assignment-operator.md)\)、メソッド呼び出し [\(\)](../../../csharp/language-reference/operators/invocation-operator.md)、[new](../../../csharp/language-reference/keywords/new.md)、[\-\-](../../../csharp/language-reference/operators/decrement-operator.md) 演算、または [\+\+](../../../csharp/language-reference/operators/increment-operator.md) 演算を表していないものです。  詳細については、「[ステートメント、式、および演算子](../../../visual-basic/reference/command-line-compiler/index.md)」を参照してください。  
  
## 使用例  
 次の例では CS0201 エラーが生成されます。これは、2 \* 3 がステートメントではなく式であるためです。  コードがコンパイルされるようにするには、式の値を変数に代入します。  
  
```  
// CS0201.cs  
public class MainClass  
{  
   public static void Main()  
   {  
      2 * 3;   // CS0201  
      // Try the following line instead.  
      //   int i = 2 * 3;  
   }  
}  
```  
  
## 使用例  
 次の例では CS0201 エラーが生成されます。これは、checked がインクリメント演算でパラメーター化されていても、checked 自体はステートメントでないためです。  
  
```  
// CS0201_b.cs  
// compile with: /target:library  
public class MyList<T>   
{  
   public void Add(T x)  
   {  
      int i = 0;  
      if ( (object)x == null)  
      {  
         checked(i++);   // CS0201  
  
         // OK  
         checked {  
            i++;   
         }  
      }  
   }  
}  
```
