<span data-ttu-id="fd027-115">アプリケーションは、他のオンプレミスアプリから自律的にアプリを開発し、クラウドでアプリのリリース、デプロイ、自動スケール、監視、トラブルシューティングを行うことができる場合に、クラウドに最適化されています。</span><span class="sxs-lookup"><span data-stu-id="fd027-115">Your application is Cloud-Optimized if you can-in an agile manner-develop your app autonomously from other on-premises apps, and then release, deploy, auto-scale, monitor, and troubleshoot your app in the cloud.</span></span>

<span data-ttu-id="fd027-116">鍵は*機敏*です。</span><span class="sxs-lookup"><span data-stu-id="fd027-116">The key is *agility*.</span></span> <span data-ttu-id="fd027-117">デプロイと運用の間の問題、開発/テスト環境の問題を最小限に抑えることができない限り、機敏性を備えたものにはなりません。</span><span class="sxs-lookup"><span data-stu-id="fd027-117">You can't ship with agility unless you reduce to an absolute minimum any deployment-to-production issues and dev/test environment issues.</span></span> <span data-ttu-id="fd027-118">コンテナー (具体的には、Docker は事実上標準的な標準) とマネージサービスは、特にこの目的のために設計されています。</span><span class="sxs-lookup"><span data-stu-id="fd027-118">Containers (specifically, Docker, as a de facto standard) and managed services were designed specifically for this purpose.</span></span>

<span data-ttu-id="fd027-119">機敏性を実現するには、クラウド内のスケーラブルなプラットフォームにリリースされる CI/CD パイプラインに基づく自動 DevOps プロセスも必要です。</span><span class="sxs-lookup"><span data-stu-id="fd027-119">To achieve agility, you also need automated DevOps processes that are based on CI/CD pipelines that release to scalable platforms in the cloud.</span></span> <span data-ttu-id="fd027-120">スケーラブルで回復力のあるクラウドプラットフォーム (Azure App Service や Azure Kubernetes Service など) にデプロイされる CI/CD プラットフォーム (Azure DevOps Services、Jenkins など) は、クラウドで機敏性を実現するための主要なテクノロジです。</span><span class="sxs-lookup"><span data-stu-id="fd027-120">CI/CD platforms (like Azure DevOps Services or Jenkins) that deploy to a scalable and resilient cloud platform (like Azure App Service or Azure Kubernetes Service) are key technologies for achieving agility in the cloud.</span></span>

- <span data-ttu-id="fd027-127">**回復力と拡張性に優れたクラウド**。</span><span class="sxs-lookup"><span data-stu-id="fd027-127">**Resilient and scalable cloud**.</span></span> <span data-ttu-id="fd027-128">クラウドは、管理、エラスティック、拡張性、回復力のあるプラットフォームを提供します。</span><span class="sxs-lookup"><span data-stu-id="fd027-128">The cloud provides a platform that is managed, elastic, scalable, and resilient.</span></span> <span data-ttu-id="fd027-129">これらの特性は、コストを向上させ、高可用性と信頼性の高いアプリケーションを継続的デリバリーで出荷するための基礎となります。</span><span class="sxs-lookup"><span data-stu-id="fd027-129">These characteristics are fundamental to gain cost improvements and ship highly available and reliable applications in a continuous delivery.</span></span> <span data-ttu-id="fd027-130">マネージデータベース、サービスとしてのマネージキャッシュ (CaaS)、管理されたストレージなどの管理されたサービスは、アプリケーションのメンテナンスコストを軽減するための基本要素です。</span><span class="sxs-lookup"><span data-stu-id="fd027-130">Managed services like managed databases, managed cache as a service (CaaS), and managed storage are fundamental pieces in alleviating the maintenance costs of your application.</span></span>

<span data-ttu-id="fd027-147">既存のアプリケーションをクラウド向けに最適化されたアプリケーションに変換することで、次のような利点を得ることができます (再設計やコーディングは不要です)。</span><span class="sxs-lookup"><span data-stu-id="fd027-147">You can get the following benefits by converting an existing application to a Cloud-Optimized application (without rearchitecting or coding):</span></span>

- <span data-ttu-id="fd027-148">**クラウドプロバイダーによって管理されるインフラストラクチャが処理されるため、コストが削減さ**れます。</span><span class="sxs-lookup"><span data-stu-id="fd027-148">**Lower costs, because the managed infrastructure is handled by the cloud provider**.</span></span> <span data-ttu-id="fd027-149">クラウド向けに最適化されたアプリケーションは、すぐに使用できる弾力性、自動スケール、高可用性を備えたクラウドのメリットを実現します。</span><span class="sxs-lookup"><span data-stu-id="fd027-149">Cloud-Optimized applications get the benefits of the cloud by using the cloud's out-of-the-box elasticity, autoscale, and high availability.</span></span> <span data-ttu-id="fd027-150">特典は、コンピューティング機能 (Vm とコンテナー) だけでなく、DBaaS、CaaS、アプリケーションが必要とするインフラストラクチャなど、クラウド内のリソースにも依存します。</span><span class="sxs-lookup"><span data-stu-id="fd027-150">Benefits are related not only to the compute features (VMs and containers), but also depend on resources in the cloud, like DBaaS, CaaS, and any infrastructure an application might needed.</span></span>

- <span data-ttu-id="fd027-151">**回復力のあるアプリケーションとインフラストラクチャ**。</span><span class="sxs-lookup"><span data-stu-id="fd027-151">**Resilient application and infrastructure**.</span></span> <span data-ttu-id="fd027-152">クラウドに移行する場合は、一時的な障害を受け入れる必要があります。クラウドでエラーが発生します。</span><span class="sxs-lookup"><span data-stu-id="fd027-152">When you migrate to the cloud, you need to embrace transient failures; failures will occur in the cloud.</span></span> <span data-ttu-id="fd027-153">また、クラウドインフラストラクチャとハードウェアは "置き換え可能" で、一時的なダウンタイムの可能性が高まります。</span><span class="sxs-lookup"><span data-stu-id="fd027-153">Also, cloud infrastructure and hardware are "replaceable," which increases opportunities for transient downtime.</span></span> <span data-ttu-id="fd027-154">同時に、内部クラウドの機能と回復性を実装する特定のアプリケーション開発手法によって、クラウドでの予期しない障害からの回復がはるかに簡単になります。</span><span class="sxs-lookup"><span data-stu-id="fd027-154">At the same time, inner cloud capabilities and certain application development techniques that implement resiliency and automate recovery make it much easier to recover from unexpected failures in the cloud.</span></span>
